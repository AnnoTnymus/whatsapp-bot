El problema

😓

Atención manual 24/7

Cada consulta de un socio o prospecto requería respuesta humana inmediata. Preguntas repetidas sobre horarios, documentación, genéticas, REPROCANN.

📋

Sin CRM ni seguimiento

Leads se perdían. No había forma de recontactar automáticamente a quien no completó la documentación o está tramitando el REPROCANN.

🔀

Proceso inconsistente

Cada operador preguntaba cosas distintas. Sin flujo estándar para validar documentación, tono de comunicación, o criterios de aceptación.

📉

Sin datos

Nulo registro de conversaciones, tiempos de respuesta, tasas de conversión. Imposible optimizar lo que no se mide.

Funcionalidades clave

💬

Atención al cliente primero

El bot prioriza responder consultas (horarios, genéticas, REPROCANN). Solo entra al flujo de afiliación cuando el usuario lo pide explícitamente.

📋

Flujo opt-in de afiliación

Saludo → nombre → conversación libre → detección de intent "quiero afiliarme" → pide DNI y REPROCANN.

👁️

Vision AI permisivo

Claude analiza DNI y REPROCANN. Si un dato no se lee, lo pide por texto — no fuerza al usuario a sacar otra foto.

⏰

Follow-ups inteligentes

Recontacta automáticamente según motivo: sin REPROCANN, tramitando, docs incompletos, inactivo.

🗃️

CRM integrado

Historial completo de cada chat. Base de miembros con estado, intentos, timestamps.

🎭

Personalidad variada

Respuestas jocosas a stickers y audios con rotación anti-repetición. Cada usuario recibe experiencia distinta.

Mejoras v4.1 — experiencia del cliente

Rediseño basado en feedback de testing real

🔁

Flujo opt-in

Antes: Hola → "mandame DNI".
Ahora: Hola → "¿En qué te ayudo?" → responde consultas → si el usuario dice "quiero afiliarme" arranca proceso.

🗣️

Tono cordial

Antes: "Boludo, mandame los papeles che".
Ahora: Cordial y profesional. "Che" solo en respuestas casuales (stickers, audios).

📸

Vision permisivo

Antes: Rechazaba fotos "borrosas" aunque se leyeran.
Ahora: Acepta docs legibles. Si falta algún dato, lo pide por texto.

🎤

Audios con humor

"Uyyy boluuu no puedo escuchar audios todavía, el jefe no me dio oídos jaja" — 5 variantes rotativas.

🎨

Stickers con personalidad

"Me faltan superpoderes para entender stickers 🦸 pero seguro estuvo re piola" — tono jocoso y útil.

🔀

Rotación anti-repetición

El bot recuerda qué respuesta usó por chat y no repite dos veces seguidas la misma variante.

Base de datos

4 tablas en Supabase con RLS y service_role bypass para el bot

👥

members

CRM principal. chat_id, nombre, estado_afiliacion, reprocann_status, fecha_alta, último contacto.

💬

conversation_history

Historial completo. Cada mensaje entrante/saliente, timestamp, tipo (texto/imagen/audio).

🔔

followups

Cola de seguimientos. motivo, proxima_notificacion, intento, status (pendiente/enviado/cancelado).

📊

bot_state

Estado conversacional. chat_id + estado actual + datos temporales del flujo de afiliación.
